# Задача 2. Геосетка и демография

**Цель:** построить H3-сетку на Красноярск, привязать к каждому гексагону оценку населения с учётом плотности застройки.

## Вход

- `data/raw/krasnoyarsk_districts_population.csv` (население по 7 районам)
- OpenStreetMap (скачивается через код)

## Выход

- **Файл 1:** `data/processed/hexagons.geojson`
  - `h3_id` — идентификатор гексагона
  - `geometry` — полигон гексагона
  - `district` — район, к которому относится центр гекса
  - `area_km2` — площадь гекса
  - `building_count` — количество зданий в гексе
  - `residential_floors_sum` — суммарная этажность жилых зданий
  - `population_estimated` — оценка населения гекса
- **Файл 2:** `data/processed/districts.geojson` — полигоны 7 районов из OSM
- **Валидация:** сумма `population_estimated` по всем гексам должна равняться 1 187 771 ± 2%

## Этапы работы

### 1. Полигоны
Через `osmnx` выкачать границы:
- города Красноярска (`admin_level=6`)
- 7 районов (`admin_level=9`)

Сохранить в GeoJSON.

### 2. H3-сетка
На площадь города наложить гексагональную сетку H3.

⚠️ **Предварительно решить с командой, какое разрешение брать** (рекомендация: res 8, гексы ~500 м радиусом).

### 3. Привязка гекса к району
Для каждого гекса определить, в каком районе находится его центр. Если пересекает границу — относится к тому району, где бóльшая часть площади.

### 4. Здания из OSM
Выкачать все объекты с тегом `building=*` в границах города.

Для каждого гекса посчитать:
- сколько зданий в нём
- у скольких указана этажность (`building:levels`)
- сумма этажности жилых (`building=apartments`, `building=residential`)

### 5. Дасиметрическое распределение
Для каждого района распределить его население между входящими в него гексами пропорционально сумме этажности жилых зданий в каждом гексе.

Формула:
```
население_гекса = население_района × (этажность_гекса / сумма_этажности_района)
```

### 6. Валидация
- Пустые гексы (только лес/промзона без жилья) должны получить ~0 населения
- Гексы центра Советского района — тысячи человек
- Проверить глазами через Folium с цветовой шкалой

## Инструменты

- [`osmnx`](https://osmnx.readthedocs.io/) — скачивание данных OSM
- [`h3`](https://h3geo.org/docs/api/indexing) — генерация гексагональной сетки
- `geopandas` — работа с геоданными (полигоны, пересечения, присоединения)
- `shapely` — геометрические операции (нижележит под geopandas)
- `folium` — визуальная проверка

## Подводные камни

- **Установка osmnx на Windows.** Зависит от GDAL, ставится через `conda`, а не `pip`, иначе боль.
- **Неравномерное покрытие OSM.** В Красноярске заполнен неравномерно — в центре все дома с этажами, на окраинах могут быть пустые теги. Это ожидаемое ограничение, упомянуть в отчёте.
- **Границы районов в OSM** могут отличаться от административных. Не все районы размечены как `admin_level=9`, может понадобиться ручная доработка.
- **Системы координат.** H3 работает с WGS84 (градусы), площадь — в метрах. Не перепутать системы координат при расчётах. Для Красноярска подходит EPSG:32646 (UTM зона 46N) для расчётов в метрах.

## Критерий готовности

В Folium открывается цветная карта с гексами, где интенсивность цвета соответствует оценке населения. Сумма сходится с итогом переписи.
