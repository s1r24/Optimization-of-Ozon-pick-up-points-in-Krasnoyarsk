# Задача 3. Коммерческая недвижимость

**Цель:** собрать датасет доступных для аренды коммерческих помещений в Красноярске с адресами и ценами.

## Вход

Ничего, кроме желания разобраться с парсингом и API.

## Выход

- **Файл:** `data/processed/commercial_realty.csv`
- **Колонки:**
  - `id` — уникальный идентификатор объявления
  - `source` — с какого сайта взято (`avito`, `cian`, `n1`)
  - `address` — адрес помещения
  - `latitude` — широта
  - `longitude` — долгота
  - `area_m2` — площадь в квадратных метрах
  - `price_rub_per_month` — арендная плата в рублях
  - `price_per_m2` — цена за квадратный метр (для сравнения)
  - `realty_type` — тип помещения (street retail, офис и т.п.)
  - `url` — ссылка на объявление
  - `scraped_at` — дата и время сбора
- **Ожидаемый объём:** 100–500 объявлений
- **Фильтр:** коммерческие помещения 20–150 м² (типичная площадь ПВЗ), только аренда

## Возможные источники

Упорядочены по сложности реализации.

### Вариант A — Авито
- **Плюсы:** самый большой выбор объявлений
- **Минусы:** официального публичного API нет, только коммерческий по договору
- **Пути:**
  - Парсинг через `requests` + `BeautifulSoup` — работает, пока Авито не поставил защиту. Может перестать работать в любой момент.
  - Selenium/Playwright — надёжнее, но медленнее
  - Есть риск блокировки IP — парсить через прокси или с задержками

### Вариант B — ЦИАН
- Ситуация похожа на Авито
- Есть партнёрский API, но он платный и для юрлиц
- Парсинг HTML — аналогично

### Вариант C — Яндекс.Недвижимость
- [Яндекс предоставляет фиды](https://yandex.ru/support/realty/) для агентств, но это не публичный API
- Парсинг возможен

### Вариант D — N1.ru, Domofond, Мир Квартир
- Региональные и агрегаторы
- В Красноярске N1 исторически силён
- Часто парсятся проще

### Вариант E — запасной, если парсинг совсем не идёт
- Ручной сбор 100 объявлений через интерфейс сайта (2–3 часа работы) — для учебного проекта сойдёт
- Взять demo-датасет с Kaggle и адаптировать под Красноярск (не идеально, но показывает владение пайплайном)

## Этапы работы

### 1. Разведка
Открыть 2–3 сайта, глянуть в **DevTools → Network**, как устроены их запросы. Часто бывает, что сайт внутри использует свой JSON-API — тогда парсить не HTML, а сразу JSON. Это гораздо проще.

### 2. Тестовый парсер
Собрать 10 объявлений с одного сайта, убедиться, что все поля вытаскиваются корректно.

### 3. Масштабирование
- Добавить пагинацию
- Задержки между запросами (2–3 секунды)
- Сохранение промежуточных результатов (если упадёт — не начинать с нуля)

### 4. Геокодация
Адреса объявлений превратить в координаты. Используйте **тот же инструмент, что участник 1** — чтобы не дублировать работу и результаты были сопоставимы.

### 5. Чистка
- Убрать дубли (одно объявление может быть на нескольких сайтах)
- Фильтровать по площади
- Привести цены к единому формату (рубли в месяц)

## Инструменты

- `requests` + `BeautifulSoup4` — база для HTML-парсинга: [документация BS4](https://www.crummy.com/software/BeautifulSoup/bs4/doc.ru/)
- `playwright` или `selenium` — если сайт грузит данные через JS: [документация Playwright](https://playwright.dev/python/)
- `fake-useragent` — чтобы менять заголовки запросов
- `pandas` — сохранение результата

## Подводные камни

### Юридический аспект
Парсинг коммерческих сайтов формально нарушает их Terms of Service. Для учебного проекта это не катастрофа, но упомянуть в отчёте честно: «данные получены парсингом публичных объявлений, для академических целей».

### Капча и блокировки
Авито и ЦИАН активно борются с парсерами. Может оказаться, что с первого раза ничего не получится — планируйте время с запасом.

### Чувствительность к дизайну сайта
Сегодня селекторы CSS работают, завтра сайт обновился — парсер сломался. Не доводите до перфекционизма, цель — один работающий срез данных.

### Этика
Ставьте задержки между запросами (2–3 сек), не долбите сервер в 100 потоков. Это хорошая инженерная культура.

## Критерий готовности

CSV с 100+ записями, все с координатами в пределах Красноярска, ценами в рублях, площадями в квадратных метрах.
